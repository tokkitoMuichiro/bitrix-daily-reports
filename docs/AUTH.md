# Вход только через Битрикс24 (AUTH_ID)

Приложение пускает сотрудников, которые открыли его **из Битрикс** (локальное приложение).  
Прямая ссылка в браузере без токена → экран «Нужен вход через Битрикс» и API отвечает 401.

## Как это работает

1. Сотрудник уже вошёл в `ammir.bitrix24.ru`
2. Открывает пункт меню **«АММИР отчёт»**
3. Битрикс открывает ваш URL в iframe и передаёт `AUTH_ID`
4. Сайт проверяет токен: `user.current`
5. Если пользователь с вашего портала — форма работает

Вебхук (`BITRIX_WEBHOOK_URL`) по-прежнему нужен серверу, чтобы читать задачи и писать на Диск.  
`AUTH_ID` — только чтобы **узнать, кто открыл сайт**.

## Настройка локального приложения в Битрикс

1. **Разработчикам** → **Другое** → **Локальное приложение** → добавить  
2. Заполните:
   - **Название:** АММИР отчёт  
   - **Путь вашего обработчика:** `https://ВАШ_ДОМЕН/`  
     (пока без HTTPS можно `http://78.24.217.87/` — для iframe лучше HTTPS)  
   - **Путь для первоначальной установки:** тот же URL или оставьте пустым  
   - **Использует только API:** нет (нужен UI)  
   - **Добавить пункт в меню:** да / **Показать в меню слева**  
3. Сохранить  
4. В левом меню включить пункт (карандаш настройки меню), если не появился сам  

Откройте пункт меню — должна загрузиться форма и строка «Вы вошли: …».

## Настройки `.env` на сервере

```env
BITRIX_PORTAL_DOMAIN=ammir.bitrix24.ru
REQUIRE_BITRIX_AUTH=1
MOCK_BITRIX=0
APP_PUBLIC_URL=https://ВАШ_ДОМЕН
```

Перезапуск:

```bash
pm2 restart stroy-otchet
```

## Локальная разработка на ПК

Чтобы открывать `http://localhost:3000` без Битрикс:

```env
MOCK_BITRIX=1
REQUIRE_BITRIX_AUTH=0
```

или только:

```env
REQUIRE_BITRIX_AUTH=0
```

## Проверка

| Действие | Ожидание |
|----------|----------|
| Открыть сайт напрямую по IP/домену | Экран «Нужен вход через Битрикс» |
| Открыть из меню Битрикс | Форма + «Вы вошли: ФИО» |
| `curl http://IP/api/tasks` без заголовков | 401 |

## Мобильное приложение

В мобильном Битрикс пункт локального приложения ищите в **Ещё → Приложения**.  
Если пункта нет — закрепите ссылку в чате **временно** (без AUTH_ID вход будет закрыт при `REQUIRE_BITRIX_AUTH=1`).

Для мобилки надёжнее открывать именно **приложение из Битрикс**, а не голый URL.  
Пока HTTPS/меню не готовы, для теста можно временно `REQUIRE_BITRIX_AUTH=0` — но это снова открытый доступ.

## Частые проблемы

- **«Загрузка приложения» бесконечно** — не вызван `BX24.installFinish()`.  
  В настройках локального приложения:
  - **Путь обработчика:** `http://ВАШ_IP/`  
  - **Путь для первоначальной установки:** `http://ВАШ_IP/install.html`  
  Затем удалите приложение и создайте заново (или переустановите).  
  Также Битрикс (HTTPS) часто **блокирует HTTP** в iframe — нужен HTTPS.
- **Белый экран / нет AUTH_ID** — приложение открыто не из Битрикс или URL неверный  
- **403 «только с портала»** — в `.env` другой `BITRIX_PORTAL_DOMAIN`  
- **Токен недействителен** — сессия истекла, откройте пункт меню заново  
- **На HTTP в iframe режут** — переходите на HTTPS (домен + certbot)
